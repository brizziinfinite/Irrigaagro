# Diário — 2026-04-28

## O que foi feito

### Fix — Precipitação não atualizava o Manejo Diário
- **Problema diagnosticado**: cliente lançou 25mm em `/precipitacoes` mas o Manejo Diário não refletia a chuva
- **Causa raiz 1**: o cron de `daily-balance` roda às 02:00 BRT — quando a chuva foi lançada depois disso, o registro do dia já existia sem os 25mm, e o recálculo automático rodava em background mas fechava o modal antes de concluir
- **Causa raiz 2**: a função `syncManagementForPivotDate` engolia erros silenciosamente (`catch { return false }`)
- **Fix aplicado** (`src/app/(app)/precipitacoes/page.tsx`):
  - Modal permanece aberto com "Atualizando manejo…" até o recálculo concluir
  - Botões Salvar e Cancelar ficam desabilitados durante o sync
  - Modal fecha automaticamente ao terminar — usuário vê o resultado pronto
- **Recálculo manual** executado via `POST /api/cron/recalculate-season` com `Authorization: Bearer` para corrigir os dados de 27/04 dos dois pivôs (25mm registrados e FC% recalculado)

### Otimização de performance — Desktop e Mobile lentos
- **Diagnóstico**: middleware (`proxy.ts`) fazia 2 chamadas ao Supabase a cada navegação de página:
  1. `supabase.auth.getUser()` — validação JWT
  2. Query em `company_members JOIN companies` — verificação de status da empresa
  - Total: 200–600ms de overhead por request **antes** de qualquer conteúdo carregar
- **Fix — Cookie de cache** (`src/proxy.ts`):
  - Cookie `co_status` (httpOnly, TTL 5 min) cacheia o status da company
  - Na maioria das navegações: zero queries ao banco no middleware
  - Cache miss (primeiro acesso ou expirado): busca no banco e grava o cookie
  - `AdminClient.tsx`: invalida `co_status` ao ativar/suspender empresa via `document.cookie = 'co_status=; max-age=0'`
- **Fix — next.config.ts**:
  - `compress: true` — brotli/gzip nas respostas (~70% menos dados no wire)
  - `optimizePackageImports: ['lucide-react', 'date-fns', 'recharts']` — tree-shaking correto, menos JS por página

### Correção de memória
- URL de produção corrigida: **irrigaagro.com.br** é o domínio ativo (não gotejo.com.br)

## Decisões técnicas

- **Cookie httpOnly + TTL 5min**: tradeoff intencional — status de empresa muda raramente (só via /admin). 5 minutos de delay máximo entre suspensão e bloqueio é aceitável. Cookie httpOnly impede leitura por JS malicioso.
- **Modal aguarda sync antes de fechar**: UX mais clara — cliente sabe que o sistema atualizou antes de navegar ao Manejo. Alternativa (fechar e atualizar async) causava confusão.
- **Não extrair recharts para dynamic imports**: recharts está muito entranhado em `manejo/page.tsx` (23 usos inline). Refatoração de alto risco sem ganho garantido. `optimizePackageImports` resolve o tree-shaking sem tocar no código.
