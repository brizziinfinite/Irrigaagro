## 2. Decisões técnicas

| Decisão | Motivo |
|---------|--------|
| Cookie + localStorage para empresa ativa | localStorage é inacessível no server; cookie é lido via `next/headers` em Server Components |
| `preferredCompanyId` opcional em `getUserCompanyOrThrow` | Sem quebrar callers existentes; cron e outros contexts não precisam da preferência |
| `farms!inner` no filtro de company | Supabase: `!inner` garante que rows sem match no join são excluídas |
| Guard na UI + RLS como defesa dupla | UI previne tentativas normais; RLS bloqueia tentativas maliciosas |

## 3. Problemas resolvidos

| Problema | Causa | Solução |
|----------|-------|---------|
| Empresa resetava para primeira em todo refresh | Sem persistência de `switchCompany` | localStorage + cookie |
| `/relatorios` expunha dados de todas as empresas | Sem `useAuth()`, queries sem filtro | company scoping via farm_ids |
| API de energia sem autenticação | Route handler sem verificação de sessão | Supabase server client + ownership check |
| Cron poderia ser manipulado para expor contextos | `listActiveManagementSeasonContexts` sem filtro | Parâmetro `companyId` opcional |
| CRUD em precipitações sem validação de pivotId | UI filtrava mas modals não validavam | Guard com `pivots.some()` |
