## 2. Decisões técnicas

| Decisão | Motivo |
|---|---|
| Tabela `rainfall_records` separada de `daily_management` | Pluviômetro independente por pivô, sem vínculo com safra |
| Leaflet com import dinâmico (`ssr: false`) | Leaflet usa `window` — SSR quebraria o build |
| CSS Leaflet importado direto no componente | Sem CSS, os tiles ficam fragmentados |
| Detecção de abas via HTML público | API do Google Sheets exige OAuth; o HTML público tem os metadados embutidos |
| SVG puro para gráfico de barras | Sem dependência extra, mais leve que Recharts para esse caso simples |

---

## 3. Problemas resolvidos

| Problema | Causa | Solução |
|---|---|---|
| "Map container is already initialized" | React Strict Mode monta componente duas vezes; `.then()` assíncrono chegava depois do cleanup | Flag `cancelled` + delete `_leaflet_id` no cleanup |
| Tiles do mapa fragmentados | CSS do Leaflet não carregado | `import 'leaflet/dist/leaflet.css'` no topo do componente |
| Google Sheets erro 400 | Planilha não estava pública | Mensagem de erro melhorada com instrução exata; planilha precisa ser pública |
| Mapa não centralizava no pivô | Guard `_leaflet_id` bloqueava reinicialização após cleanup | Removido guard redundante, mantido só o `cancelled` flag |
