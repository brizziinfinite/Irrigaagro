## Decisões técnicas

- **PTF vs tabela FAO-56**: PTF Saxton & Rawls dá valores específicos para a granulometria real — mais preciso que tabela tabelada. FAO-56 mantido como fallback quando agricultor não tem laudo.
- **`field_capacity_percent` do banco para cor**: não recalcular ao vivo (raiz pode ter crescido → CTA diferente → % diferente). Usar sempre o valor gravado pelo cron — mesma fonte que dashboard.
- **Culturas padrão editáveis**: ao invés de forçar duplicar → editar, agora editar direto cria a cópia internamente. UX limpa sem proliferação de "(cópia)".
- **Multi-cultura sem refatorar balanço**: `daily_management` já é por `season_id` → múltiplas safras no mesmo pivô geram múltiplos registros automaticamente. Só precisou de `area_ha` em `seasons` para o rateio financeiro.
- **Rateio igualitário como fallback**: quando `area_ha` não preenchido, divide igual entre safras do pivô. Aviso amarelo indica ao agricultor que deve preencher para rateio preciso.

## Problemas resolvidos

- **Caixa d'água laranja vs verde**: `resolvePivotStatus` recalculava `adcMm/ctaMm` com raiz do DAS atual (maior) → % menor → cor errada. Fix: usar `lastMgmt.field_capacity_percent`.
- **Save de cultura travando**: `.update().select().single()` sob RLS retorna 0 linhas → trava. Removido `.select().single()`.
- **Cron 03/05 não rodou**: causa desconhecida na Vercel. Dado estava na planilha Plugfield. Recuperado manualmente. Alerta adicionado para próximas falhas.
- **Cards mobile cortando**: `display: flex` sem `minWidth: 0` em filhos de grid → overflow. Fix padrão aplicado.
- **Turbopack cache**: componente `SoilParametersInput` atualizado não aparecia. Fix: `rm -rf .next/cache` + reiniciar.
