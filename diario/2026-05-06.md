## Decisões técnicas

- **`sem_credenciais` retorna cache existente** em vez de erro 422 — evita "tela branca" para cliente que ainda não configurou API key mas já tem dados históricos
- **`date-fns` em vez de `.toLocaleDateString`** — consistência com AgroManager, formato controlado, sem dependência de locale do browser
- **`useRefreshNdvi` genérico** para pivot e talhão — antes tinha dois paths separados (um usando o hook, outro inline sem estado)
- **Diagnóstico `caindoForte`** — limiar -15% para separar "queda leve" de "declínio acelerado" (padrão do AgroManager)

## Problemas resolvidos

- Talhão refresh não mostrava loading nem capturava erro (era `supabase.functions.invoke` solto sem `await`)
- `fmtData` sem ano na tabela de leituras — difícil comparar datas de anos diferentes no histórico
- Diagnóstico não usava `nuvens` — NDVI baixo por nuvens era interpretado como problema da lavoura
