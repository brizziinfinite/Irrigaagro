# Diário de Desenvolvimento — 2026-03-30

## O que foi feito

### Feature: ETo com Rs NASA + painel de calibração super admin (`feature/eto-rs-nasa`)

1. **`src/lib/weather-fetch.ts`** — adicionada função `fetchRsFromNASA(lat, lon, dateISO)` que busca apenas `ALLSKY_SFC_SW_DWN` da NASA POWER para uma data específica, retornando `number | null` (MJ/m²/dia). `WeatherDay` ganhou campo opcional `evapoPlugfield?: number | null`. `fetchFromPlugfield` agora captura `d.evapo` e retorna no campo `evapoPlugfield`.

2. **`src/app/api/cron/ingest-weather/route.ts`** — cron atualizado:
   - Após buscar dados do Plugfield/Sheets/Open-Meteo, tenta buscar Rs da NASA POWER para a mesma data D-1
   - Se NASA retorna Rs válido → usa `rgForEto = rsNasaMJ / 0.0864` (W/m²) e `rsSource = 'nasa'`
   - Se NASA falha → mantém `rsSource = 'plugfield_fallback'` e usa radiação original
   - Upsert no `weather_data` agora inclui `eto_plugfield_mm` (valor `evapo` bruto do Plugfield) e `rs_source`

3. **`supabase/migrations/weather_data_eto_plugfield.sql`** — migration adicionando colunas `eto_plugfield_mm numeric(6,2)` e `rs_source text DEFAULT 'unknown'` à tabela `weather_data`. Aplicada com sucesso via MCP.

4. **`src/types/database.ts`** — `WeatherData` e `WeatherDataInsert` atualizados com `eto_plugfield_mm` e `rs_source`.

5. **`src/lib/super-admin.ts`** — novo utilitário que verifica `NEXT_PUBLIC_SUPER_ADMIN_EMAILS` para identificar super admins no frontend.

6. **`.env.local`** — adicionado `NEXT_PUBLIC_SUPER_ADMIN_EMAILS=fazbrizzi@gmail.com`.

7. **`src/app/(app)/diagnostico-pivo/page.tsx`** — painel "Comparativo ETo — Calibração" visível apenas para super admin:
   - Componente `EtoCalibrationPanel` com tabela: Data | Rs fonte | ETo FAO-56 | ETo Plugfield | Diferença (mm) | Diferença (%)
   - Linhas coloridas: verde (<0.5mm), âmbar (<1.5mm), vermelho (≥1.5mm)
   - Média do período no rodapé
   - Badge "SUPER ADMIN" âmbar no header da seção
   - Guarda contra objeto inline em deps de useEffect (usa `etoStationId` estabilizado)

## Decisões técnicas

- Rs em MJ/m²/dia (unidade NASA) → W/m² via `÷ 0.0864` para passar ao `calcETo` existente
- `evapoPlugfield` no `WeatherDay` interface (opcional, `undefined` para fontes não-Plugfield) — sem quebrar callers existentes
- `rs_source` default `'unknown'` para rows antigas; novas rows terão `'nasa'` ou `'plugfield_fallback'`
- Super admin identificado no cliente via `NEXT_PUBLIC_SUPER_ADMIN_EMAILS` (simples, sem backend extra)

## Problemas resolvidos

- Nenhum erro TypeScript ou de build
- Dependências do `useEffect` do painel super admin estabilizadas via `etoStationId` (evita loop)
