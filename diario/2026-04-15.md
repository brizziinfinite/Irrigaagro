## Decisões técnicas

- `crossesMidnight(start, end)` — retorna true se `toMin(end) < toMin(start)`. Simples e eficaz.
- `handleClearAll()` reseta o `sectorGrids` para `emptyEntry()` em todos os dias/setores.
- `cellInput` para `type='time'` retorna wrapper flex em vez do input direto (Chrome ignora `textAlign`).
- Kc continua sendo calculado internamente (necessário para ETc e balanço hídrico) — só saiu da UI.
- Ondas do gráfico usam posição absoluta calculada via média dos dados + mapeamento linear para pixels (margem top=16, bottom=30, height=320).

## Problemas resolvidos

- Input `type="time"` não centraliza com `textAlign: center` no Chrome/Safari — resolvido com wrapper flex.
- Replicar colocava Setor D no mesmo dia mesmo quando C terminava no dia seguinte — corrigido com `crossesMidnight()`.
- Tooltip do mapa cortado — corrigido com `direction: 'top'` e padding extra no fitBounds.
- CC aparecia em posição errada no gráfico (83% visual em vez de 100%) — corrigido com domínio Y dinâmico.
