### 4. Fix PRINCIPAL: ETo subestimada — Plugfield evapo vs FAO-56
**Diagnóstico:**
- O sensor Plugfield tem campo `evapo` (evaporação de tanque) ~2.27mm
- O sistema priorizava esse valor bruto como ETo em vez de calcular Penman-Monteith FAO-56
- FAO-56 com as variáveis climáticas do mesmo sensor = ~3.3mm (30-44% maior)
- Também: altitude 600m hardcoded no `calcETo` local do ingest-weather vs 500m no `water-balance.ts`

**Correções (`ingest-weather/route.ts`):**
- Removida a prioridade de `evapo` como ETo
- Sempre calcular FAO-56 com: tmax, tmin, humidity, wind, solar_radiation_wm2
- Campo `evapo` continua salvo em `eto_plugfield_mm` como referência histórica
- Altitude corrigida: 600m → 500m

**Novo endpoint `admin/fix-eto-history`:**
- Recalcula FAO-56 para todos os registros históricos com `rs_source = 'plugfield_evapo'`
- Grava novo `rs_source = 'plugfield_fao56'` para rastreabilidade
- Preserva `eto_plugfield_mm` (valor bruto original)
- Protegido por CRON_SECRET, suporta `?dry_run=true`

**Execução da correção histórica:**
- 7 registros corrigidos (13/04 a 20/04): ETo médio +1.01mm/dia

- FC% atualizado com ETo correto

## Decisões técnicas

- **Não usar evapo do tanque como ETo**: subestima por natureza (tanque menor que superfície FAO-56). O sensor tem valor agronômico mas não substitui o cálculo Penman-Monteith.
- **rs_source = 'plugfield_fao56'**: novo valor para identificar registros onde dados vieram do Plugfield mas ETo foi calculado FAO-56. Mantém rastreabilidade do histórico.
- **Altitude 500m**: padronizado em todo o sistema (ingest-weather + water-balance.ts).
