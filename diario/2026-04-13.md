# Diário de Desenvolvimento — 2026-04-13

## O que foi feito

### 1. Correção crítica: fonte de chuva (Plugfield removido)
- **Problema**: O sistema usava dados de chuva do sensor Plugfield (`weather_data.rainfall_mm`) como fallback quando não havia lançamento manual. Isso causava erros graves no balanço hídrico (ex: dia 18/03 com 49mm do Plugfield vs 0mm correto).
- **Regra definida**: Chuva = **exclusivamente** `rainfall_records` (lançamentos manuais via página Precipitações). Se não há lançamento = 0mm. Plugfield NUNCA é fonte de chuva.
- **Fix aplicado** em 3 arquivos de rota:
  - `src/app/api/seasons/recalculate/route.ts`
  - `src/app/api/cron/recalculate-season/route.ts`
  - `src/app/api/admin/recalculate-season/route.ts`
  - Removido `?? climateSnapshot.rainfall_mm` de todos os payloads `rainfall_mm`

## Decisões técnicas

- **Plugfield como fonte de chuva**: definitivamente descartado. Sensor acumulado não é confiável para lançamentos diários.
- **ETo diferente Plugfield/sistema**: esperado e aceitável — Plugfield usa outra fórmula; sistema usa Penman-Monteith FAO-56 × 0.82.
- **Timeline sempre visível**: mais útil que escondida no accordion — o gráfico é o coração visual do manejo.
- **N8N image builder**: geração de imagem HTML via Code Node para envio no WhatsApp como imagem renderizada.
