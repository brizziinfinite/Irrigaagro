# Diário de Desenvolvimento — 2026-04-27

## O que foi feito

### feat(auth): Sistema de aprovação de clientes

Implementação completa do fluxo de onboarding controlado:

1. **Migration Supabase** — coluna `status text DEFAULT 'pending'` em `companies`
   - Valores: `pending` | `active` | `suspended`
   - Clientes existentes migrados automaticamente para `active`
   - Índice em `companies(status)` para performance

2. **proxy.ts** — verificação de status integrada ao fluxo de autenticação
   - `pending` / `suspended` → redireciona para `/aguardando`
   - `/admin` → bloqueado para não super-admins (verifica `NEXT_PUBLIC_SUPER_ADMIN_EMAILS`)
   - `/aguardando` sempre acessível (não entra em loop)

3. **Página `/aguardando`** — tela informativa para clientes pendentes
   - Mostra e-mail da conta
   - Polling a cada 15s: detecta aprovação e redireciona automaticamente para `/dashboard`
   - Botão "Sair da conta"

4. **Página `/admin`** (super-admin only)
   - KPIs: total / pendentes / ativos
   - Filtros por status com badge de contagem
   - Lista de clientes com e-mail do owner, data de cadastro, nº de membros
   - Botões: Ativar / Suspender / Reativar
   - Feedback inline após cada ação

5. **API `/api/admin/notify-activation`** — e-mail via Resend para cliente ao ser ativado
6. **API `/api/admin/notify-new-signup`** — e-mail para Brizzi quando novo cliente se cadastra
7. **Sidebar** — seção "Admin" com link `/admin` visível apenas para `fazbrizzi@gmail.com`
8. **Trigger Supabase** `on_new_user_notify` — chama webhook após novo cadastro

---

## Decisões técnicas

- **proxy.ts em vez de middleware.ts**: o projeto usa `proxy.ts` (padrão Next.js 16 com Turbopack) — integrei a lógica ali ao invés de criar um arquivo separado
- **Polling no cliente (15s)**: mais simples que WebSocket/realtime para este caso de uso; aceita latência de até 15s para aprovação
- **Clientes existentes → `active`**: migration faz UPDATE imediato para não bloquear ninguém que já estava usando
- **WEBHOOK_SECRET**: Supabase não permite `ALTER DATABASE` via API; trigger chama webhook sem autenticação por enquanto (rota interna, risco baixo)
