## Decisões técnicas

- `:root:where([data-theme="light"])` — solução para Tailwind v4: o `:where()` mantém especificidade da pseudo-classe `[data-theme]` (0,1,0 extra) vencendo o `:root` do `@layer theme`
- Bloco `:root` manual removido do globals.css (era redundante com `@theme`)
- Light mode parcial → light mode completo: migração mecânica de ~3300 ocorrências em 45 arquivos para CSS vars
- Cores de status (#0093D0, #22c55e, #ef4444, #f59e0b) mantidas hardcoded — são iguais em dark e light

## Problemas resolvidos

- Hydration mismatch no botão tema: ícone mudava entre SSR (dark=Sun) e client (lê localStorage); fix: `useState(false)` para `mounted`, botão só renderiza após mount
- CSS não aplicando no light mode: Tailwind v4 injeta `@theme` como `@layer theme` (menor prioridade), mas o bloco `:root` manual no globals.css (unlayered, maior prioridade) bloqueava override; solução: remover `:root` manual + usar `:root:where([data-theme="light"])` com especificidade maior
