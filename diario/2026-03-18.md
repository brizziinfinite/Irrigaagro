# Diário — 2026-03-18

## O que foi feito

### Feature: Extração de Conta de Energia com IA

1. **Migration SQL** — `supabase/migrations/energy_bills.sql`
   - Tabela `energy_bills` com todos os campos do plano
   - Colunas geradas: `reserved_percent`, `reactive_percent`, `month`
   - RLS via pivot → farm → company_members
   - Constraint UNIQUE (pivot_id, reference_month)

2. **Tipos TypeScript** — `src/types/database.ts`
   - Interface `EnergyBill` e `EnergyBillInsert`

3. **API Route** — `src/app/api/extract-energy-bill/route.ts`
   - Aceita multipart/form-data: `file`, `pivot_id`, `irrigated_mm_ha` (opcional)
   - LLM primária: Gemini 2.0 Flash Lite via REST direto (sem sdk extra)
   - Fallback: GPT-4o-mini (OpenAI)
   - Validação manual (sem zod para evitar dependência extra)
   - UPSERT no Supabase via service_role key
   - Calcula KPIs: reactivePct, reservedPct, peakCost, costPerMmHa

4. **Página /relatorios** — nova seção de energia
   - Seletor de pivô
   - Campo lâmina irrigada/mês (para custo/mm/ha)
   - Dropzone (clique ou arraste) para PDF/imagem
   - Botão "Extrair com IA" com loading state
   - KPIs semáforo do último mês: custo/mm/ha, reativa%, HR%, ponta, demanda
   - Gráfico SVG histórico: barras reativa% + linha custo/mm/ha
   - Tabela histórico com status semáforo por linha

5. **N8N Workflow** — `n8n/energy-bill-whatsapp.json`
   - 9 nodes: Webhook → IF mídia → Download → Map phone → Extract → IF success → Format → Reply
   - Mapeamento telefone→pivô via variável de ambiente `PHONE_TO_PIVOT_MAP`
   - Resposta formatada com todos os KPIs e emojis semáforo

6. **.env.local** — adicionadas keys de placeholder:
   - `GEMINI_API_KEY`
   - `OPENAI_API_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`

## Decisões técnicas

- **Gemini via REST direto** (sem `@google/generative-ai`) para não adicionar dependência — mesma lógica para OpenAI (sem sdk)
- **Validação manual** em vez de zod: projeto já sem zod, não vale adicionar dependência só para isso
- **Colunas geradas** no SQL para `reserved_percent` e `reactive_percent` — calculadas automaticamente pelo banco
- **Modelo OpenAI**: usei `gpt-4o-mini` (não "gpt-5 mini" que não existe) — o plano dizia "GPT-5 Mini" mas esse modelo não existe ainda; usar o melhor mini disponível

## Problemas resolvidos

- TypeScript error: Supabase retorna `farms` como array em joins; corrigido com `Array.isArray` check no tipo local `PivotRow`
- Modelo Gemini: `gemini-2.0-flash-lite` é o ID correto do modelo (não `gemini-2.5-flash-lite` que pode não existir)

## Pendências — O QUE FAZER ANTES DA PRÓXIMA SESSÃO

### ✅ OBRIGATÓRIO para a feature de energia funcionar

**Passo 1 — Aplicar a migration no Supabase**
1. Abrir https://supabase.com/dashboard → projeto IrrigaAgro
2. Menu lateral → **SQL Editor**
3. Colar e executar o conteúdo do arquivo: `supabase/migrations/energy_bills.sql`
4. Verificar se a tabela `energy_bills` apareceu em Table Editor

**Passo 2 — Preencher as API keys no `.env.local`**

Abrir o arquivo `irrigaagro-v2/.env.local` e substituir os placeholders:

- `GEMINI_API_KEY` → https://aistudio.google.com/apikey (gratuito)
- `OPENAI_API_KEY` → https://platform.openai.com/api-keys
- `SUPABASE_SERVICE_ROLE_KEY` → Supabase Dashboard → Settings → API → campo `service_role` (clicar em "Reveal")

**Passo 3 — Testar o upload**
```bash
cd irrigaagro-v2
npm run dev
```
Acessar http://localhost:3000/relatorios → seção "Conta de Energia" → fazer upload de uma foto/PDF de conta Energisa.

### 🔧 OPCIONAL — N8N (WhatsApp)

Só fazer quando a fase de upload no app estiver funcionando:
1. N8N → menu Workflows → Import → selecionar `n8n/energy-bill-whatsapp.json`
2. Criar credential "Evolution API Key" (HTTP Header Auth: `apikey` = sua key da Evolution)
3. Configurar variáveis de ambiente no N8N (Settings → Variables):
   - `IRRIGAAGRO_APP_URL` = URL do app em produção (ou http://localhost:3000)
   - `EVOLUTION_API_URL` = URL base da Evolution API
   - `PHONE_TO_PIVOT_MAP` = JSON mapeando telefones para pivot_ids, ex: `{"5518999998888": "uuid-do-pivo"}`
4. Ativar o workflow e configurar o webhook na Evolution API
