### 3. Diagnóstico crítico descoberto
**Classes Tailwind adicionadas dinamicamente não são geradas no bundle** (Tailwind v4). Classes como `p-4`, `p-5`, `p-6`, `sm:grid-cols-4` adicionadas em edições desta sessão não estavam no scan estático inicial — resultavam em `padding: 0px`, `gridTemplateColumns: repeat(2, ...)` no browser. Solução: **sempre usar `style` inline para propriedades novas críticas** em vez de classes Tailwind não confirmadas.

## Decisões técnicas

| Decisão | Motivo |
|---|---|
| `overflow: clip` em vez de `hidden` no PivotTable | `hidden` cria novo contexto de scroll bloqueando `overflowX: auto` interno |
| Padding inline em vez de Tailwind nos novos componentes | Tailwind v4 só gera classes presentes no scan estático; classes novas precisam de `style` inline ou adicionar ao `tailwind.config` safelist |
| `sm` (640px) como breakpoint para 4 colunas KPI | Breakpoint baseado no viewport (960px CSS real), não no container (695px) |
| Nomes dos pivôs com glow em vez de contagem genérica | Informação acionável — usuário sabe exatamente qual pivô agir |
| Ícone `DecisionCard` oculto no mobile (`hidden sm:flex`) | 260px CSS não comporta ícone 52px + gap + texto "IRRIGAR HOJE: SIM" |

---

## Problemas resolvidos

- **`overflow: hidden` bloqueando scroll interno** → `overflow: clip`
- **Playwright DPR 1.5** → viewport CSS = physicalPx / 1.5 (260 CSS = 390 físicos)
- **Classes Tailwind não geradas** → padding zero, grid 2 colunas em desktop — fix: style inline
- **DecisionCard inutilizável** → ativado, responsivo, com dados reais
